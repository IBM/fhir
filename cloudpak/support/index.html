Support policy details

Versioning Model

IBM FHIR Server uses a semantic versioning format consisting of a three-digit version representing the < major >.< minor >.< patch > release versions:

  • Major - A new major version may include backward incompatible changes and minor and patch level changes.
  • Minor – A minor version may include new backwards compatible functionality, deprecate any public API function and include patch level changes.
  • Patch – A patch version may include backwards compatible bug fixes and vulnerability updates.

IBM FHIR Server provides fix updates for releases designated as a long term support release (LTSR). The long term support release provides a minimum of 12 months of support. The update version, type, general availability date and the availability of fixes is shown in the table below.
